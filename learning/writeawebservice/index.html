    Write A Web Service

    To write a Web Service, you will do the following:

    1. Write initialization logic, such as:
      • Start IModelHost
      • Initialize Logging
      • Configure FeatureGates
    2. Write the operations of the service, such as:
      • Access Elements
      • Execute ECSQL queries
    3. Expose the operations of the service as RpcInterfaces:
      • Define one or more RpcInterfaces.
      • Implement the RpcInterfaces.
      • Serve the RpcInterfaces to clients.
    4. Write a simple Web server
    5. Package and deploy to the Web

    Note that a service typically does not open an iModel on its own initiative. Instead, normally, a client of the service will ask the service to open an iModel, and then the client will pass the resulting IModelToken to methods of the service. The service should therefore always initialize the IModelReadRpcInterface.

    A service may also need to:

    • Synchronize with iModelHub
