    Write An Interactive App

    To write an interactive iModeljs app, first divide the app's operations into two categories:

    • Presentation = frontend
    • iModel access = backend.

    1. Write the Frontend

    See the frontend learning articles.

    2. Write the Backend

    Some apps can use an already deployed Web service. In that case, there is no need to write a backend. Just write the frontend's main to point to the remote service.

    To write a customized backend, do the following:

    1. Write initialization logic, such as:
      • Start IModelHost
      • Initialize Logging
      • Configure FeatureGates
    2. Write the operations of the backend, such as:
      • Access Elements
      • Execute ECSQL queries
      • Modify the iModel
    3. Expose the operations of the backend as RpcInterfaces.
      • Define one or more RpcInterfaces.
      • Implement the RpcInterfaces.
      • Configure the RpcInterfaces.

    Note that an app backend typically does not open an iModel on its own initiative. Instead, normally, the app frontend will ask the backend to open an iModel, and then the frontend will pass the resulting IModelToken to methods of the backend. The backend should therefore always initialize the IModelReadRpcInterface.

    A backend may need to:

    • Synchronize with iModelHub

    3. Tailor, Package, and Deploy the App

    An interactive app must be tailored to match each desired configuration and sometimes each target platform. It must then be packaged and deployed using appropriate tools. For more information, see:

    • Interactive Web app
    • Interactive desktop app
    • Interactive mobile app
