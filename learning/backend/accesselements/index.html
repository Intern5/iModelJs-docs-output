    Accessing Elements in an IModelDb

    An iModelJs Element is an in-memory representation of a BIS Element. An Element object provides easy access to a set of properties and Aspects.

    IModelDb.elements represents the collection of Elements in an iModel.

    Every iModel contains a single "root" element that is the parent of all partitions. It is accessed by a special function:

    Example:

        const root = iModel.elements.getRootSubject();

    You can look up all other elements in various ways. As described in BIS Element Fundamentals, an element is identified by a unique ID and may also have a Code and/or a FederationGuid. If you know one of these identifiers, you can look up an element very efficiently. The IModelDb.Elements.getElement method makes that easy.

    Example:

        const el = iModel.elements.getElement(root.id);
    assert.deepEqual(el.id, root.id);

    const el2 = iModel.elements.getElement(root.code);
    assert.deepEqual(el2.id, root.id);
    assert.deepEqual(el2.code, root.code);

    You can discover an element's ID by using ECSQL to query other properties or even using spatial queries
