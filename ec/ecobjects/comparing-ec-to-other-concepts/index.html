Comparing EC to other concepts

🔗.NET Type System

The type system of ECObjects is loosely patterned after the .NET Type System (including its concept of "custom attributes"). Notably, the .NET concept of "object references" is replaced with NavigationProperties for expressing relationships among objects and StructProperties for expressing embedded objects.

🔗UML

ECObjects has many similarities with a limited subset of UML concepts. A standard mapping of EC concepts into UML is defined to facilitate as a graphical syntax for an ECSchema. Note that it is not possible to map from arbitrary UML into ECSchema because ECSchema is intentionally more restrictive than UML. Generally ECSchemas can be considered as UML Packages and many other key concepts as sterotyped UML Classes, with ECRelationshipClasses being like UML Association Classes.

Table mapping some EC concepts to UML concepts, as well as other common corresponding concepts:

EC UML Other/General
Set of ECSchemas Depicted in a Package Diagram Conceptual Schema
ECSchema Depicted in a Package Namespace (and all of the concepts defined in that namespace)
ECClass Depicted as Box in a Class Diagram Concept, Class, Entity Type
ECClass.BaseClasses Depicted as Specializations/Generalizations Multiple inheritance with overriding of attributes/properties
ECRelationshipClass Depicted as Association, Aggregation, or Composition among Boxes. May need its own box if the ECRelationshipClass has ECProperties of its own. Relationship, Link, Link Table in RDBMS
ECRelationshipClass.RelationshipStrength Composition; Aggregation; Association Cascading delete; sharing; pointing-to
ECRelationshipConstraints and multiplicity Map to start and end boxes in UML and standard UML depiction of cardinality N/A
ECProperty Depicted as an Attribute in the Class Box Attribute, Member, Property
ECArrayProperty Depicted as an Attribute with [ ] Like a simple ordered array of primitives in C++
ECStructProperty Depicted as the name of the ECStructClass as the datatype Like an embedded struct in C++
ECStructArrayProperty Similar to ECStructProperty, with the addition of [ ] Like an ordered array in C++, where the members are structs
ECCustomAttribute N/A Just like .NET Custom Attributes. Extensible self-describing metadata for any purpose.

🔗XML Schema and JSON Schema

It would be misguided to compare an ECSchema to an XML Schema (XSD) or JSON Schema, because those two are not languages for defining entity types in an object system, but for validating hierarchical documents.

Although there is some overlap between these concepts, an ECSchema is able to define more complex relationships between entity types that are not possible to define within XML and JSON Schemas.

For example, a simple primitive entity would probably look similar in an ECSchema, XML Schema and Json Schema, but as more information and concepts are added, like navigation properties or relationship constraints, it becomes apparent that the concepts have a different scope.

🔗CSDL

ECSchema is comparable to the CSDL (Conceptual Schema Definition Language) of the .NET Entity Framework and the CSDL (Common Schema Definition Language) of OData, though both of those are data modeling languages rather than information modeling languages. ECObjects 2.0 predates both of those data modeling languages.

CSDL has some concepts that do not exist in ECSchema as of today. For example, CSDL breaks apart EntityTypes from EntitySets. An EntityType describes the structure of an object (like a Person), while an EntitySet refers to a source of entities. For example, the EntitySet "People" would provide a collection of entities that are of the EntityType "Person". Think of EntityType describing the structure of a table, while an EntitySet would be an actual named table.

🔗EXPRESS

ECObjects could also be compared to the EXPRESS data modeling language (the language used to model IFC.)
