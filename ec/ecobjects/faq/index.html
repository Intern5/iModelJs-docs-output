FAQ

This section elaborates on some of the design decisions behind ECObjects.

🔗Why have separate ECPrimitiveProperty, ECStructProperty, ECPrimitiveArrayProperty and ECStructArrayProperty types

There are structural differences in each of these flavors of properties, and having them as separate elements will make handling of them more convenient.

🔗Why have separate ECEntityClass, ECStructClass and ECCustomAttribute class types?

The reason for use of a different type for these ECObjects concepts is that their definitions are disjoint and different rules applied to how each type of class can be used. Splitting them into different types makes it obvious how an individual class is supposed to be used and helps when applying rules to a set of classes.
