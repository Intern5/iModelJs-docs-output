Specification Proposals

🔗EC 3.2 Proposals

The following proposals are actively undergoing review and discussion so they may change as the proposal is refined.

🔗Modification to KindOfQuantity

🔗Reason

Enable references to Units and Formats defined in schemas, see Adding Units to an ECSchema

🔗Proposed Changes

PersistenceUnit Modified

  • Required
  • A fully qualified name of a Unit. A default format is always provided in the absence of any PresentationUnits.

Example:

u:M

PresentationFormats Modified

  • Optional
  • A string list of Format Strings using a ; as a separator.

Example:

f:DefaultRealU<6>[u:M];f:DefaultRealU<4>[u:CM|centimeter(s)];f:DefaultRealU<2>[u:MM|millimeters(s)]

🔗Enhancements to ECEnumeration

🔗Reason

The ECEnumerator unlike a most of other items in an ECSchema does not contain an ECName that is unique within the containing ECEnumeration.

🔗Proposed Changes

The following proposed changes are to the existing ECEnumerator specification.

Name Addition

  • Required
  • Valid ECName
  • Must be unique within the ECEnumeration

Value Modified

  • Required
  • Must be unique within the ECEnumeration

DisplayLabel Modified

  • Optional
  • A localized display label that is used instead of the name in a GUI.
  • If not set the Name will be used.

Description Addition

  • Optional
  • A localized description for the enumerator.

Xml:

Old:

<ECEnumerator value="1" displayLabel="First One"/>

New:

<ECEnumerator name="FirstOne" value="1" displayLabel="First One"
        description="Represents the second thing because we always start counting at zero" />

Json:

Old:

{
"value": 1,
"label": "First One"
}

New:

{
"name": "FirstOne",
"value": 1,
"label": "First One",
"description": "Represents the second thing because we always start counting at zero"
}

🔗Add Units as a first-class concept in a Schema

🔗Reason

The motivation for adding the ability to define Units, and everything that goes with them, within an ECSchema as a first-class concept is to allow the iModel to not be dependent on any hard-coded Units, as it is now.

🔗Proposed Changes

Each of the following are new first-class concepts in an ECSchema.

Unit
Name
  • Required
  • Valid ECName
Display Label
  • Optional
Description
  • Optional
Phenomenon
  • Required
  • Fully qualified name of the Phenomenon {Schema name}.{Phenomenon Name}
Unit System
  • Required
  • Fully qualified name of the Unit System {Schema name}.{Unit System Name}
Definition
  • Required
Numerator
  • Optional
  • Default: 1.0
Denominator
  • Optional
  • Default: 1.0
Offset
  • Optional
  • Default: 0.0

Xml:

<Unit typeName="MM" displayLabel="Millimeter" description="A unit defining the millimeter metric unit of length"
        phenomenon="Length" unitSystem="Metric" definition="[MILLI]*M" numerator="1.0" denominator="1.0" offset="0.0"/>

Json:

{
"schemaItemType": "Unit",
"schema": "ExampleSchema",
"name": "MM",
"label": "Millimeter",
"description": "A unit defining the millimeter metric unit of length",
"phenomenon": "ExampleSchema.Length",
"unitSystem": "ExampleSchema.Metric",
"definition": "[MILLI]*M",
"numerator": 1.0,
"denominator": 1.0,
"offset": 0.0
}
InvertedUnit

An InvertedUnit is a a specific type of Unit that describes the inverse of a single Unit whose dimensional derivation is unit-less.

Name
  • Required
  • ECName
Display Label
  • Optional
Description
  • Optional
InvertsUnit
  • Required
  • Fully qualified name of the Unit this is the inverse of, {Schema name}.{Unit Name}
Unit System
  • Required
  • Fully qualified name of the Unit System {Schema name}.{Unit System Name}

Xml:

<InvertedUnit typeName="HORIZONTAL_PER_VERTICAL" displayLabel="Horizontal/Vertical" description="A unit representing run over rise" 
        unitSystem="INTERNATIONAL" invertsUnit="VERTICAL_PER_HORIZONTAL"/>

Json:

{
"schemaItemType": "InvertedUnit",
"schema": "ExampleSchema",
"name": "HORIZONTAL_PER_VERTICAL",
"label": "Horizontal/Vertical",
"description": "A unit representing run over rise",
"unitSystem": "ExampleSchema.INTERNATIONAL",
"invertsUnit": "ExampleSchema.VERTICAL_PER_HORIZONTAL",
}
Constant

A Constant is a specific type of Unit that represents a number.

Name
  • Required
  • ECName
Display Label
  • Optional
Description
  • Optional
Phenomenon
  • Required
  • Fully qualified name of the Phenomenon {Schema name}.{Phenomenon Name}
Definition
  • Required
Numerator
  • Optional
  • Default: 1.0
Denominator
  • Optional
  • Default: 1.0

Xml:

<Constant typeName="DECI" displayLabel="" description="" definition="ONE" numerator="1.0" denominator="10.0" 
        phenomenon="METRIC_PREFIX" />

Json:

{
"schemaItemType": "Constant",
"schema": "ExampleSchema",
"name": "DECI",
"label": "",
"description": "",
"definition": "ONE",
"numerator": "1.0",
"denominator": "10.0",
"phenomenon": "ExampleSchema.METRIC_PREFIX",
}

Name
  • Required
  • Valid ECName
DisplayLabel
  • Optional
Description
  • Optional
Definition
  • Required

    • AREA: LENGTH(2)

Xml:

<Phenomenon typeName="AREA" displayLabel="Area" description="" definition="LENGTH(2)" />

Json:

{
"schemaItemType": "Phenomenon",
"name": "AREA",
"label": "Area",
"description": "",
"definition": "LENGTH(2)"
}

Name
  • Required
  • Valid ECName
DisplayLabel
  • Optional
Description
  • Optional

Xml:

<UnitSystem typeName="IMPERIAL" displayLabel="Imperial" description="Units of measure from the british imperial empire" />

Json:

{
"schemaItemType": "UnitSystem",
"name": "IMPERIAL",
"label": "Imperial",
"description": "Units of measure from the british imperial empire",
}

🔗Add Formats as a first-class concept in a Schema

🔗Format

Name
  • Required
  • ECName
DisplayLabel
  • Optional
Description
  • Optional
RoundFactor
  • Double
  • Optional
  • Default: 0.0
Type
  • String
  • Required
  • Options:
    • decimal
    • fractional
    • scientific
    • station
Precision
  • int
  • Required
  • Options
    • Type is decimal, scientific or station, 0 - 12
    • Type is fractional, 1|2|4|8|16|32|64|128|256
ScientificType
  • string
  • Required if type is scientific
  • Options
    • standard
    • normal
    • engineering
ShowSignOption
  • String
  • Optional
  • Default: onlyNegative
  • Options:
    • noSign
    • onlyNegative
    • signAlways
    • negativeParentheses
FormatTraits
  • String | Array
    • Array: An array of strings
      • Only used in the ECJSON.
      • Ex:
        [
  "leadZeroes",
  "trailZeroes"
]
    • String: A string list using a ,, ; or | as a separator, providing an alternative way to describe the above array.
      • Ex: trailZeroes|showUnitName
  • Optional
  • Options:
    • trailZeroes
      • Indicates that one or more insignificant zeroes are to be added after the last digit of the fraction.
    • keepSingleZero
      • Indicates that the fractional part of the number is required when the fraction is zero.
    • zeroEmpty
      • Indicates that zero value should be presented by an empty string.
    • keepDecimalPoint
      • Indicates that the decimal point is should be presented when the fraction is zero.
    • applyRounding
      • Use the rounding factor.
    • fractionDash
      • Use a dash between integer and fraction instead of a space: 3-1/4 rather than 3 1/4.
    • showUnitLabel
      • Indicates that the numeric expression should be followed by the unit label.
      • Note: If the label is desired in front of the numeric expression, set the prependUnitLabel trait in addition to this trait.
    • prependUnitLabel
      • Indicates the position of the label shifts from being after the numeric expression to in front of it. The showUnitLabel FormatTrait needs to be set in order for the Unit label to be shown, but the location of the label is overriden.
    • use1000Separator
      • Indicates that thousands in the integer part of the number should be separated by a special char (. or,).
    • exponentOnlyNegative
      • Indicates that if an exponent value is positive to not include a +. By default a sign, + or -, is always shown.
DecimalSeparator
  • Optional
  • Default: Based on current locale
    • Example: US .
    • Example: European ,
ThousandsSeparator
  • Optional
  • Default: Based on current locale
    • Example: US: ,
    • Example: European .
UOMSeparator
  • Optional
  • Default:
  • Defines the separator between the magnitude and the Unit
    • Example: If the spacer is set to -, the Quantity is now displayed as 4-in. If the spacer is the default, the Quantity is 4 in.
StationSeparator
  • Optional
  • Default: +
StationOffsetSize
  • int
    • Positive integer greater than 0.
  • Required when type is station
  • Represents the magnitude of the StationOffset
    • Example: StationOffsetSize of 2, in base 10, represents an offset of 100.

🔗Sub-Elements

Composite (0..1)

Xml:

<Format typeName="AmerMYFI4" displayLabel="myfi4" description="" roundFactor="0.0" type="fractional"
          signOption="onlyNegative" formatTraits="leadZeroes|trailZeroes" precision="4"
          fracBarType="diagonal" decSeparator="." thousandSeparator=","
          uomSeparator=" ">
    <Composite spacer="-">
      <Unit label="mile(s)">MILE</Unit>
      <Unit label="yrd(s)">YRD</Unit>
      <Unit label="'">FT</Unit>
      <Unit label="\"">IN</Unit>
    </Composite>
</Format>

{
  "schemaItemType": "Format",
  "name": "AmerMYFI4",
  "label": "myfi4",
  "description": "",
  "roundFactor": 0.0,
  "type": "fractional",
  "signOption": "onlyNegative",
  "formatTraits": "leadZeroes|trailZeroes",
  "precision": 4,
  "fractBarType": "diagonal",
  "decSeparator": ".",
  "thousandSeparator": ",",
  "uomSeparator": " ",

    "spacer": "-",

      {
        "name": "MILE",
        "label": "mile(s)"
      },

        "unitName": "YRD",

      },
      {
        "unitName": "FT",
        "unitLabel": "'",
      },
      {
        "unitName": "IN",
        "unitLabel": "\"",
      },
    ],
  }
}

🔗Composite

A Composite defines additional information about a format and whether the magnitude should be split and display using multiple Units.

spacer
  • Optional
  • Default:
  • Is used as the spacer between the composite spec components
    • Example: Spacer set to -;
        5 mile(s)-400 yrd(s)
      Spacer set to default, ;
        5 mile(s) 400 yrd(s)
Sub-Elements

Unit (1..4)

Unit has:

  • Attributes
    • label
      • Optional
  • Value:
    • Fully qualified name of the Unit

Xml:

<Composite spacer="-">
    <Unit label="mile(s)">MILE</Unit>
    <Unit label="yrd(s)">YRD</Unit>
    <Unit label="'">FT</Unit>
</Composite>

Json:

{
  "spacer": "-",
  "units": [
    {
      "name": "MILE",
      "label": "mile(s)"
    },
    {
      "unitName": "YRD",
      "unitLabel": "yrd(s)",
    },
    {
      "unitName": "FT",
      "unitLabel": "'",
    },
  ],
}

🔗Format String

The Format String is a short string-based representation of a Format, which allows overriding certain key properties of a Format.

Syntax

The below represents the string literal syntax. All italized values are to be replaced with appropriate values.

formatName<precision> [unitName|unitLabel]`[*unitName*|*unitLabel*][`*unitName*`|`*unitLabel*`][*unitName*|*unitLabel*]`

formatName

  • Required
  • An valid fully qualified ECName referring to a Format

<precision>

  • Optional override of the precision defined in the Format.

The order of the Unit overrides, [unitName|unitLabel], corresponds directly to the order they are used within the Composite. If no Units are defined within the Format, the Units defined in the Format String are used. Otherwise, all unit names must match the Unit defined in the Format and if a label is provided it will override the label defined in the Format.

Note: If there are Units already defined within the format the number of Unit overrides must match exactly, and the Units must be the same.

unitName

  • Optional
  • Name of a unit.
    • The name must be a fully qualified ECName.

unitLabel

  • Optional
  • A string that overrides the label of the input Unit.
    • Disallowed characters to use within the label override are; ] and ;. If either are desired, a new Format is required.
