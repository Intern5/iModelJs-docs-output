  • Units

    •
  • ECObjects - An Object Oriented language for modeling, manipulating and persisting information with implementations in many programming languages
  • ECO - An abreviation for ECObjects
  • EC - Another abreviation for ECObjects
  • ECSchema - Defines a named grouping of named elements describing an information model or pare of one. Used as both a container and a namespace.
  • ECClass - The generic term for any named element which describes an object.
  • EntityClass - A specialization of ECClass which defines an individually addressable object
  • MixinClass - A specialization of ECClass which defines an interface
  • StructClass - A specialization of ECClass which defines a complex property type
  • CustomAttributeClass - A specialization of ECClass which defines metadata which can be applied to other ec elements.
  • RelationshipClass - A specialization of ECClass which defines the connection between two entities
  • Class Modifier - Identifies an ECClass as abstract or sealed.
  • RelationshipConstraint - Part of a RelationshipClass definition which identifies what entities it can connect
  • Source - The name for one RelationshipConstraint of a relationship, normal traversal is from source to target. Required
  • Target - The name for one RelationshipConstraint of a relationship, normal traversal is from source to target. Required
  • Constraint Classes - Part of a RelationshipConstraint which defines the entity or mixin classes which are acceptable for that endpoint. Required
  • Abstract Constraint Class - Part of a RelationshipConstraint which defines a common base base class when more than one constraint class is defined. Required if more than one constraint class defined.
  • Polymorphic - An attribute of a RelationshipConstraint which defines if classes which derive from the constraint classes are acceptable.
  • Role Label - Part of a RelationshipConstraint which provides a short, user friendly label used when displaying the connection between the two related entities.
  • Multiplicity - Part of a RelationshipConstraint which defines the number of times a given entity instance may be used as an endpoint for a class of relationship.
  • Strength - Part of a RelationshipClass which defines the ownership relationship between the two endpoint instances
  • StrengthDirection - Part of a RelationshipClass, defines the direction the Strength applies in. Forwards is from Source to Target and Backwards is from Target to Source
  • ECProperty - The generic term for any property which may be defined within an ECClass
  • PrimitiveProperty - A specialization of ECProperty which defines a property of a primitive type
  • StructProperty - A specialization of ECProperty which defines a property whose type is defined by a StructClass
  • ArrayProperty - A specialization of ECProperty which defines an array property of a primitive type
  • StructArrayProperty - A specialization of ECProperty which defines an array property whose type is defined by a StructClass
  • NavigationProperty - A specialization of ECProperty which defines a property whose type is the Id of a related instance
  • KindOfQuantity - An item of ECSchema which describes a type of measured value including unit, relative error and default formatting.
  • FormatUnitSpecification - The combination of a unit and a display format
  • Persistence FUS - Part of a KindOfQuantity, a FormatUnitSpecification which describes the persistence of a value.
  • Presentation FUS - Part of a KindOfQuantity, a FormatUnitSpecification which describes the presentation of a value.
  • Relative Error - Part of a KindOfQuantity, a ratio of measurement error by measurement magnitude.
