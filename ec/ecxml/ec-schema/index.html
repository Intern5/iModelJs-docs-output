ECSchemaXML Elements In Detail

EC is implemented independent of XML but the XML representation serves as a good way to describe the important persistent parts of each EC element.

ECSchemaXML Version URNs:

  • 1.X uses the XML namespace with URN: http://www.bentley.com/schemas/1.X/ECXML
  • 2.0 uses the XML namespace with URN: http://www.bentley.com/schemas/Bentley.ECXML.2.0
  • 3.X uses the XML namespace with URN: http://www.bentley.com/schemas/Bentley.ECXML.3.X

An XSD exists for each version to help when editing the XML directly.

🔗ECSchema

The ECSchema element is the root element in the ECSchemaXML XML document because it is the container for all other EC elements.

🔗Attributes

schemaName Required

  • Must be a valid ECName

alias Required

version Required

  • 3-part version number following the pattern RR.ww.mm.

description Optional

displayLabel Optional

  • Default is the name of the ECSchema

🔗Sub-elements

ECSchemaReference – Multiplicity (0..*)

ECCustomAttributes – Multiplicity (0..1)

ECEntityClass – Multiplicity (0..*)

ECStructClass – Multiplicity (0..*)

ECCustomAttributeClass – Multiplicity (0..*)

ECRelationshipClass – Multiplicity (0..*)

ECEnumeration – Multiplicity (0..*)

KindOfQuantity – Multiplicity (0..*)

🔗Example

<ECSchema schemaName="MyCompany_HumanResources" alias="hr" version="01.00.00" description="MyCompany’s human resources schema" xmlns="<http://www.bentley.com/schemas/Bentley.ECXML.3.0>" >

🔗ECSchemaReference

This allows one ECSchema to refer to others. Circular references are not supported and will result in a failure to load the schemas with cyclic references.

🔗Attributes

name Required

  • Must match the name of the referenced schema.

version Required

alias Required

🔗Example

<ECSchemaReference name="EditorCustomAttributes" version="01.00.00" alias="beca" />

🔗ECClass

There are 4 types of ECClass in ECObjects; ECEntityClass, ECStructClass, ECCustomAttributeClass and ECRelationshipClass. This section will cover what is common among all ECClass types.

🔗Common Class Attributes

typeName Required

  • Must be a valid ECName

description Optional

displayLabel Optional

  • Default is the typeName of this ECClass

modifier Optional

  • Default is None
  • Supported values:
    • None
    • Abstract
    • Sealed

🔗Common sub-elements

ECProperty – Multiplicity (0..*)

ECStructProperty – Multiplicity (0..*)

ECArrayProperty – Multiplicity (0..*)

ECStructArrayProperty – Multiplicity (0..*)

ECCustomAttributes – Multiplicity (0..1)

BaseClass – Multiplicity (0..*)

🔗ECEntityClass

🔗Sub-elements

ECNavigationProperty – Multiplicity (0..*)

Example:

<ECEntityClass typeName="Employee" description="Holds personal employee data" modifier="Sealed">
  <BaseClass>Person</BaseClass>
  <ECProperty propertyName="EmployeeNo" typeName="int" />
  <ECProperty propertyName="Status" typeName="EmployeeStatus" />
  <ECProperty propertyName="Name" typeName="string" />
  <ECStructProperty propertyName="Addr" typeName="Address" />
  <ECArrayProperty propertyName="Hobbies" typeName="string" minOccurs="0" maxOccurs="unbounded" />
  <ECStructArrayProperty propertyName="PriorPositions" typeName="EmploymentPosition" minOccurs="0" maxOccurs="unbounded" />
  <ECNavigationProperty propertyName="Boss" relationshipName="EmployeeReportsToBossMan" direction="forward" />
</ECEntityClass>

🔗Mixin

Mixins are a specialization of ECEntityClasses and the way to denote an ECEntityClass as a mixin is to add the IsMixin CustomAttribute.

Note: Mixins are always an ECEntityClass with a modifier of Abstract so it is unnecessary to set the modifier attribute since it will be ignored.

Example:

<ECEntityClass typeName="EmployeeMixin">
  <ECCustomAttributes>
    <IsMixin xmlns="CoreCustomAttributes.01.00">
      <AppliesToEntityClass>Employee</AppliesToEntityClass>
    </IsMixin>
  </ECCustomAttributes>
</ECEntityClass>

🔗ECStructClass

Example:

<ECStructClass typeName="Address">
  <ECProperty propertyName="Street" typeName="string" />
  <ECProperty propertyName="City" typeName="string" />
  <ECProperty propertyName="State" typeName="string" />
  <ECProperty propertyName="Zip" typeName="int" />
</ECStructClass>

🔗ECCustomAttributeClass

appliesTo Required

  • A string containing a list of supported vales using a ,, ; or | as a separator.
  • See CustomAttribute Container Types for supported values.

Example:

<ECCustomAttributeClass typeName="DateTimeInfo" appliesTo="PrimitiveProperty|ArrayProperty">
  <ECProperty propertyName="IsUTC" typeName="boolean" />
  <ECProperty propertyName="IncludesDate" typeName="boolean" />
</ECCustomAttributeClass>

🔗ECRelationshipClass

strength Required

  • Supported values:
    • Embedding
    • Referencing
    • Holding

strengthDirection Required

  • Supported values:
    • Forward
    • Backward

🔗Sub-elements

BaseClass – Multiplicity (0..1)

Source – Multiplicity (1..1)

  • Follows the format ECRelationshipConstraint

Target – Multiplicity (1..1)

  • Follows the format ECRelationshipConstraint

Example:

<ECRelationshipClass typeName="RoomContents" strength="Embedding" strengthDirection="Forward" modifier="Sealed">
  <Source multiplicity="(1..1)" polymorphic="true" roleLabel="contains">
    <Class class="Room" />
  </Source>
  <Target multiplicity="(0..*)" polymorphic="false" roleLabel="contained by" >
    <Class class="Desk" />
  </Target>
</ECRelationshipClass>

🔗ECRelationshipConstraint (Source and Target)

polymorphic Required

roleLabel Required

multiplicity Required

abstractConstraint Conditionally Optional

  • Fully qualified class name, {schema name}.{class name}.
  • Required only when the there is more than one constraint class and the ECRelationshipClass has no base classes.

🔗Sub-elements

Class: the ECClass on this end of the relationship – Multiplicity (1..*)

ECCustomAttributes – Multiplicity (0..1)

Multiple class elements are supported on Target, but only a single class element is supported on Source.

Example:

<Source multiplicity="(1..*)" polymorphic="true" roleLabel="Source constraint" abstractConstraint="">
  <Class class="" />
  <Class class="" />
</Source>

🔗ECEnumeration

typeName Required

  • Must be a valid ECName

description Optional

displayLabel Optional

backingTypeName Required

  • Supported values:
    • int
    • string

isStrict Optional

  • String representation boolean
  • Default: true

🔗Sub-elements

ECEnumerator – Multiplicity (0..*)

Example:

<ECEnumeration typeName="PrimaryColors" backingTypeName="int" description="The primary colors" displayLabel="Primary Colors." isStrict="true">  
  <ECEnumerator value="1" displayLabel="Red" />
  <ECEnumerator value="2" displayLabel="Green" />
  <ECEnumerator value="3" displayLabel="Blue" />
</ECEnumeration>

🔗ECEnumerators

value Required

  • Either a string or a string representation of a numnber. Depending on the backingTypeName of the ECEnumeration.

displayLabel Optional

Example:

<ECEnumerator value="3" displayLabel="Blue" />

🔗KindOfQuantity

typeName Required

  • Must be a valid ECName

description Optional

displayLabel Optional

persistenceUnit Required

precision optional

presentationUnits optional

  • A ; separated list of presentation units.

Example:

<KindOfQuantity typeName="DIAMETER\_SMALL" displayLabel="Diameter Small" persistenceUnit="MM" precision="3" presentationUnits="IN;MM;CM"/>

🔗PropertyCategory

typeName Required

  • Must be a valid ECName

displayLabel Optional

description Optional

priority Required

Example:

<PropertyCategory typeName="" displayLabel="" description="" priority="5">

🔗ECProperty

🔗Common ECProperty attributes

name Required

  • Must be a valid ECName

description Optional

displayLabel Optional

readOnly Optional

  • A string representing a boolean value.

category Optional

  • Fully qualified name {alias}.{property category name}.

priority Optional

  • A string representing an integer.

customAttributes Optional

  • See CustomAttributes

🔗ECPrimitiveProperty

typeName Required

  • One of the 10 primitive types or the full name of an enumeration {alias}.{enumeration name}.

extendedTypeName Optional

minimumValue Optional

  • A string representing a long, double, or integer

maxmimumValue Optional

  • A string representing a long, double, or integer

minimumLength Optional

  • A string representing an integer

maximumLength Optional

  • A string representing an integer

kindOfQuantity Optional

  • Name of KindOfQuantity or fully qualified name {alias}.{KindOfQuantity name} if the KindOfQuantity is in a referenced ECSchema.

Example:

<ECProperty propertyName="EmployeeNo" typeName="int" minimumValue="5" maximumValue="15"/>

🔗ECArrayProperty

typeName Required

  • One of the 10 primitive types or the full name of an enumeration {alias}.{enumeration name}.

extendedTypeName Required

minimumValue Optional

  • A string representing a long, double, or integer.

maxmimumValue Optional

  • A string representing a long, double, or integer.

minimumLength Optional

  • A string representing an integer.

maximumLength Optional

  • A string representing an integer.

kindOfQuantity Optional

  • Name of KindOfQuantity or fully qualified name {alias}.{KindOfQuantity name} if the KindOfQuantity is in a referenced ECSchema.

minOccurs Optional

  • A string representing an integer.

maxOccurs Optional

Example:

<ECArrayProperty propertyName="Hobbies" typeName="string" minOccurs="0" maxOccurs="unbounded" />

🔗ECStructProperty

typeName Required

  • Name of an ECStructClass or fully qualified name {alias}.{struct class name} if the class is in a referenced ECSchema.

Example:

<ECStructProperty propertyName="Addr" typeName="Address" />

🔗ECStructArrayProperty

typeName Required

  • Name of an ECStructClass or fully qualified name {alias}.{struct class name} if the class is in a referenced ECSchema.

minOccurs Optional

maxOccurs Optional

Example:

<ECStructArrayProperty propertyName="Addresses" typeName="Address" minOccurs="0" maxOccurs="unbounded" />

🔗ECNavigationProperty

relationshipName Required

  • Name of the ECRelationshipClass.
    • Note: If the class is in a reference schema a fully qualified name, {alias}.{relationship class name}, is required.

direction Required

  • Supported values:
    • "forward"
    • "backward"

Example:

<ECNavigationProperty propertyName="Boss" relationshipName="EmployeeReportsToBossMan" direction="forward" />

🔗ECCustomAttributes

Contains the custom attribute instances applied to the container that is parent to this node.

🔗Sub-elements

Instances of a type defined by an ECCustomAttribute ECClass, in ECInstanceXML format.

🔗Example

Below is the same example Employee class as seen previously, except the ECClass as a whole and one of its ECProperties have extra metadata.

<ECEntityClass typeName="Employee" description="Holds personal employee data">
  <BaseClass>Person</BaseClass>

  <ECCustomAttributes>
    <HRAppData>
      <AccessLevel>2</AccessLevel>
      <Abbreviation>Emp</Abbreviation>
    </HRAppData>
  </ECCustomAttributes>

  <ECProperty propertyName="EmployeeNo" typeName="int" />
    <ECCustomAttributes>
      <HRAppData>
        <AccessLevel>3</AccessLevel>
        <Abbreviation>EmpNo.</Abbreviation>
      </HRAppData>
    </ECCustomAttributes>
  </ECProperty>

  <ECProperty propertyName="Name" typeName="string" />
  <ECStructProperty propertyName="Addr" typeName="Address" />
  <ECArrayProperty propertyName="Hobbies" typeName="string" minOccurs="0" maxOccurs="unbounded" />
  <ECStructArrayProperty propertyName="PriorPositions" typeName="EmploymentPosition" minOccurs="0" maxOccurs="unbounded" />
</ECEntityClass>
