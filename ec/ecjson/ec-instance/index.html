ECDB Wire JSON Format for Instances

This document describes the ECDB Wire format. ECDB also has an internal FileFormat which uses similar JSON, but the two formats may change independently, the Wire Format is what will be used across platforms.

🔗JSONInstance

An instance is a JSON object which follows a certain pattern and has a set of special properties.

Example:

{
  "$ECInstanceId" : [StringValue],
  "$ECClassKey" : [StringValue],
  "PropertyName1" : [ECJsonValue],
  "PropertyName2" : [ECJsonValue]
}
  • $ECInstanceId: Unique identifier for the current instance
  • $ECClassKey: Identifier of the ECClass for this instance. Format is: SchemaName.ClassName. So for a class Wall in schema House this would be: House.Wall

🔗StringValue

Represents a simple string.

Example:

{
  "$ECInstanceId" : "test013",
  "$ECInstanceLabel" : "Contains a point",
}

🔗ECJSONValue

Can be one of the following:

  • NullValue
  • PrimitiveValue
  • StructValue
  • ArrayValue

🔗NullValue

Use null, not "null" Example instance containing a property set to null

Example:

{
  "$ECInstanceId" : "test013",
  "$ECInstanceLabel" : "Contains a point",
  "TestPoint" : null,
}

🔗PrimitiveValue

A PrimitiveType is one of the following:

  • Boolean
    • Either true or false
  • Binary
    • Contains a base64 string of binary data
  • Integer
    • Simple integer number
  • Long
    • Simple long number
  • Double
    • Double number
  • DateTime
    • Contains a date in ISO format
  • String
    • Contains a string
  • Point3d
    • Contains a three dimensional point

Examples:

Binary : "WW91IGZvdW5kIGEgc2VjcmV0LiBIZXJlLCBoYXZlIGEgZnJlZSBjb29raWUh"
Integer: 2
Long: "3483721653455"
Double: 3.5
DateTime: "2013-01-08T08:12:04.523" (ISO string)
Point3d: {"x" : 1.1, "y" : 2.2, "z" : 3.3} (JSON object value)
String: "Test"

🔗StructValue

See Point3d for an example

🔗ArrayValue

Can be seen as a set of PrimitiveValues/ECJSONValue Example instance with LastTemperatures property being an array of integer values

Example:

{
  "$ECInstanceId" : "test013",
  "$ECInstanceLabel" : "Contains a point",
  "LastTemperatures" : [ 43, 52, 43, 49 ],
}

🔗Point3d

A Point3d is similar to a StructValue but is a supported primitive type.

Example:

{
  "x" : 1.1,
  "y" : 2.2,
  "z" : 3.3
}

Example instance containing a Point3d:

{
  "$ECInstanceId" : "test013",
  "$ECInstanceLabel" : "Contains a point",
  "Origin" : {"x" : 1.1, "y" : 2.2, "z" : 3.3},
}

🔗NavigationProperty

Used to navigate instances.

Example:

{
  "$ECInstanceId" : "test013",
  "$ECInstanceLabel" : "Contains a navigation property",
  "TestNavigationproperty" : {
    "id" : "test-12",
    "relECClassId" : "data-142"
  }
}

This is an example instance which contains all the types described in this document Used properties and their type:

Property Type

AllPartsValidAngles MotorAnglesStructArray

AttachedPart NavigationProperty
Size IntValue
ProductionCosts LongValue
CenterOfRotation Point3d
