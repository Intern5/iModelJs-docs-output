ECSchemaJSON Format

There are two supported JSON formats for describing an ECSchema and the items within it. The first is an ECSchemaJSON format and the second SchemaItemJSON format.

🔗ECSchemaJSON vs. SchemaItemJSON

An ECSchema has always been described in its entirety in one XML file using the ECSchemaXML format. This ability, to define the entire ECSchema, is consistent in the ECSchemaJSON format; an entire ECSchema can be defined within a single JSON object. However, with the addition of the SchemaItemJSON format there is a new way to describe parts of the EC Information Model outside the context of the ECSchema which contains it. The SchemaItemJSON is not to replace or add anything new an ECSchema, but rather provide another way to describe the parts of an ECSchema as a way to optimize for use in web, or other scenarios, where size of data is essential.

The SchemaItemJSON definition goes into more detail on the potential use cases and highlights the benefits of the format.

🔗ECSchema

$schema Required

  • JSON Type: String
  • Contains the url to the JSON Schema describing the desired spec.
    • i.e "https://dev.bentley.com/json_schemas/ec/31/draft-01/ecschema"

name Required

  • JSON Type: String

version Required

  • JSON Type: String
  • 3-part version number following the pattern RR.ww.mm.

alias Required

  • JSON Type: String

label Optional

  • JSON Type: String

description Optional

  • JSON Type: String

references Optional

  • JSON Type: Array
  • An array of objects describing an ECSchema reference. Each object contains the following two properties;
    • name Required
      • JSON Type: String
    • version Required
      • JSON Type: String
      • 3-part version number following the pattern RR.ww.mm.

customAttributes Optional

  • JSON Type: Array
  • An array of CustomAttribute objects. Each CustomAttribute object must follow the ECJSON Instance format (ECDb wire format).

items Optional

  • JSON Type: Object
  • Contains objects of type Schema Items.
    • Note: The SchemaItems that are within a schema do not need to contain a "name" property because it is the key of the object. It also does not need to contain a "schema" or "schemaVersion" as those are defined by the schema they are within.

Example:

{
  "$schema": "https://dev.bentley.com/json_schemas/ec/31/draft-01/ecschema",
  "name": "ExampleSchema",
  "version": "1.0.0",
  "alias": "ex",
  "label": "Example Schema",
  "description": "An ECSchema to show an example of an ECJson format.",
  "references": [
    {
      "name": "CoreCustomAttributes",
      "version": "1.0.0"
    }
  ],
  "customAttributes": [
    {
      "className": "CoreCustomAttributes.HiddenSchema"
    }
  ],
  "items": {
    "TestClass": {
    ...
    },
    ...
  }
}

🔗Schema Item

The SchemaItemJSON format defined below serves two purposes. The first is to represent any top-level component of an ECSchema (i.e. ECClass, ECEnumeration, KindOfQuantity, etc.) within an ECSchemaJSON object. The second is to allow any top-level component of an ECSchema to be defined outside of the context of its holding ECSchema.

Defining a Schema Item outside the context of its ECSchema is not intended, and should not be used, to change the definition defined within the ECSchema. Rather, provide a way to describe the same information defined within the ECSchema in a more compact way when the entire ECSchema is not needed.

There is a slight difference in the SchemaItemJSON format depending on whether it is within an ECSchemaJSON object or by itself. There are 4 JSON properties that may change; $schema, schema, schemaVersion and name. These properties are used when a Schema Item is outside of its ECSchema in order to provide information about the ECSchema it is within.

🔗Common properties to all Schema Items

The following properties are in all Schema Items, ECClasses, ECEnumeration, KindOfQuantity, and PropertyCategory

$schema Conditionally Required

  • JSON Type: String
  • Contains the url to the JSON Schema describing the desired spec. i.e "https://dev.bentley.com/json_schemas/ec/31/draft-01/schemaitem"
    • Required only when the Schema Item is not used in the context of an ECSchemaJSON object.

schema Conditionally Required

  • JSON Type: String
  • Required only when the Schema Item is not used in the context of an ECSchemaJSON object.

schemaVersion Optional

  • JSON Type: String
  • A 3-part version number following the pattern RR.ww.mm.
  • Optional unless there is a scenario where you may have more than one version of the same schema.

name Conditionally Required

  • JSON Type: String
  • Required only when the Schema Item is not used in the context of an ECSchemaJSON object.

schemaItemType Required

  • JSON Type: String
  • Supported values:
    • EntityClass
    • Mixin
    • RelationshipClass
    • StructClass
    • CustomAttributeClass
    • KindOfQuantity
    • Enumeration
    • PropertyCategory

label Optional

  • JSON Type: String

description Optional

  • JSON Type: String

🔗Common ECClass Properties

The following properties are in all ECClass Types:

  • ECEntityClass
  • Mixin
  • ECStructClass
  • ECRelationshipClass
  • ECCustomAttributeClass

modifier Optional

  • JSON Type: String
  • Default: "none"
  • Supported values:
    • none
    • abstract
    • sealed

baseClass Optional

  • JSON Type: String
  • Fully qualified class name {schema name}.{class name}

properties Optional

  • JSON Type: Array
  • An array of ECProperty objects.

customAttributes Optional

  • JSON Type: Array
  • An array of CustomAttribute objects. The CustomAttributes objects should following the ECJSON Instance format (ECDb wire format).

🔗ECEntityClass

mixins Optional

  • JSON Type: Array
  • An array of fully qualified class names, {schema name}.{class name}.

Example:

{
  "$schema": "https://dev.bentley.com/json_schemas/ec/31/draft-01/schemaitem",
  "schema": "ExampleSchema",
  "schemaVersion": "1.0.0",
  "schemaItemType": "EntityClass",
  "name": "ExampleEntity",
  "modifier": "sealed",
  "label": "ExampleEntity",
  "description": "An example entity class.",
  "baseClass": "ExampleSchema.ExampleBaseEntity",
  "mixins": [
    "ExampleSchema.ExampleMixin",
    "ExampleSchema.ExampleMixin2"
  ]
}

🔗Mixin

Note: Mixins always have an ECClass modifier of Abstract, making it unnecessary to set the modifier property.

appliesTo Required

  • JSON Type: String
  • Fully qualified class name, {schema name}.{class name}.

Example:

{
  "$schema": "https://dev.bentley.com/json_schemas/ec/31/draft-01/schemaitem",
  "schema": "ExampleSchema",
  "schemaVersion": "1.0.0",
  "schemaItemType": "Mixin",
  "name": "ExampleMixin2",
  "baseClass": "ExampleSchema.ExampleMixin",
  "appliesTo": "ExampleSchema.ExampleEntity"
}

🔗ECStructClass

Example:

{
"schemaItemType": "StructClass",
"name": "ExampleStruct",
"modifier": "sealed",
"properties": [
  {
    "propertyType": "PrimitiveArrayProperty",
    "name": "ExamplePrimitiveArray",
    "typeName": "ExampleSchema.ExampleEnumeration",
    "minOccurs": 7,
    "maxOccurs": 20
  }
]
}

🔗ECCustomAttributeClass

appliesTo Required

  • JSON Type: String
  • Must be one of the CustomAttributeContainerTypes
  • Supports a list applies to types by using a ,, ; or | as a separator.

Example:

{
  "$schema": "https://dev.bentley.com/json_schemas/ec/31/draft-01/schemaitem",
  "schema": "ExampleSchema",
  "schemaVersion": "1.0.0",
  "schemaItemType": "CustomAttributeClass",
  "name": "ExampleCustomAttribute",
  "modifier": "sealed",
  "appliesTo": "Schema, AnyProperty"
}

🔗ECRelationshipClass

strength Required

  • JSON Type: String
  • Supported values:
    • embedding
    • referencing
    • holding

strengthDirection Required

  • JSON Type: String
  • Supported values:
    • forward
    • backward

source Required

  • JSON Type: Object
  • The object must the format of an ECRelationshipConstraint

target Required

  • JSON Type: Object
  • The object must the format of an ECRelationshipConstraint

Example:

{
  "$schema": "https://dev.bentley.com/json_schemas/ec/31/draft-01/schemaitem",
  "schema": "ExampleSchema",
  "schemaVersion": "1.0.0",
  "schemaItemType": "RelationshipClass",
  "name": "ExampleRelationship",
  "modifier": "sealed",
  "strength": "embedding",
  "strengthDirection": "forward",
  "source": {
    ...
  },
  "target": {
    ...
  }
}

🔗ECRelationshipConstraint

multiplicity Required

  • JSON Type: String
  • The multiplicity of this end of the relationship in the format (0..1).

roleLabel Required

  • JSON Type: String

polymorphic Required

  • JSON Type: Boolean

abstractConstraint Conditionally Required

  • JSON Type: String
  • Fully qualified class name, {schema name}.{class name}.
  • Required only when the there is more than one constraint class and the ECRelationshipClass has no base classes.

constraintClasses Required

  • JSON Type: Array
  • An array of fully qualified class names, {schema name}.{class name}.

customAttributes Optional

  • JSON Type: Array
  • An array of CustomAttribute objects. The CustomAttributes objects should following the ECJSON Instance format (ECDb wire format).

Example :

{
  "multiplicity": "(0..1)",
  "roleLabel": "source roleLabel",
  "polymorphic": true,
  "abstractConstraint": "ExampleSchema.SourceClass1",
  "constraintClasses": [
    "ExampleSchema.SourceClass1",
    "ExampleSchema.SourceClass2"
  ]
}

🔗ECEnumeration

backingTypeName Required

  • JSON Type: String
  • Supported values:
    • int
    • string

isStrict Required

  • JSON Type: Boolean

enumerators Required

  • JSON Type: Array
  • An array of objects. The objects must be the format of an ECEnumerator

Example:

{
  "$schema": "https://dev.bentley.com/json_schemas/ec/31/draft-01/schemaitem",
  "schemaItemType": "Enumeration",
  "name": "ExampleEnumeration",
  "schema": "ExampleSchema",
  "backingTypeName": "int",
  "isStrict": true,
  "enumerators": [
    {
      ...
    },
    ...
  ]
}

🔗ECEnumerator

value Required

  • JSON Type: String | Number
  • The type differs based on the backingTypeName of the ECEnumeration

label Optional

  • JSON Type: String

Example:

{
  "value": 1,
  "label": "None"
}

🔗KindOfQuantity

persistenceUnit Required

  • JSON Type: Object
  • The object must be the Unit Format.

presentationUnits Required

  • JSON Type: Array
  • The objects within the array must be the Unit Format.

precision Required

  • JSON Type: Number

🔗Unit Format

unit Required

  • JSON Type: String

format Required

  • JSON Type: String

Example:

{
  "$schema": "https://dev.bentley.com/json_schemas/ec/31/draft-01/schemaitem",
  "schemaItemType": "KindOfQuantity",
  "schema": "ExampleSchema",
  "name": "ExampleKoQ",
  "precision": 3,
  "persistenceUnit": {
    "unit": "MM",
    "format": "DefaultReal"
  },
  "presentationUnits": [
    {
      "unit": "IN",
      "format": "DefaultReal"
    }
    {
      "unit": "MM",
      "format": "DefaultReal"
    },
    {
      "unit": "CM",
      "format": "DefaultReal"
    }
  ]
}

🔗PropertyCategory

priority Required

  • JSON Type: Number

Example:

{
  "$schema": "https://dev.bentley.com/json_schemas/ec/31/draft-01/schemaitem",
  "schemaItemType": "PropertyCategory",
  "name": "ExamplePropertyCategory",
  "priority": 5
}

🔗ECProperty

🔗Common ECProperty properties

name Required

  • JSON Type: String

propertyType Required

  • JSON Type: String
  • Supported values
    • PrimitiveProperty
    • StructProperty
    • PrimitiveArrayProperty
    • StructArrayProperty
    • NavigationProperty

description Optional

  • JSON Type: String

label Optional

  • JSON Type: String

readOnly Optional

  • JSON Type: Boolean

category Optional

  • JSON Type: String
  • Fully qualified name {schema name}.{property category name}.

priority Optional

  • JSON Type: Number

customAttributes Optional

  • JSON Type: Array
  • An array of CustomAttribute objects. The CustomAttributes objects should following the ECJSON Instance format (ECDb wire format).

inherited Optional

  • JSON Type: Boolean
  • Identifies whether this property is from a base class. This should only be true if the property is defined in a base class and not overriden in the current class.

🔗ECPrimitiveProperty

typeName Required

  • JSON Type: String
  • One of the 10 primitive types or a fully qualified enumeration name {schema name}.{enumeration name}.

extendedTypeName Optional

  • JSON Type: String

minValue Optional

  • JSON Type: Number

maxValue Optional

  • JSON Type: Number

minLength Optional

  • JSON Type: Number

maxLength Optional

  • JSON Type: Number

kindOfQuantity Optional

  • JSON Type: String
  • Fully qualified name {schema name}.{KindOfQuantity name}.

Example:

{
  "propertyType": "PrimitiveProperty",
  "name": "ExamplePrimitiveProp",
  "description": "An example primitive property",
  "label": "Example Primitive Property",
  "readOnly": true,
  "category": "ExampleSchema.ExamplePropertyCategory",
  "typeName": "string",
  "extendedTypeName": "Json",
  "minLength": 12,
  "maxLength": 200,
  "kindOfQuantity": "ExampleSchema.ExampleKoQ",
  "customAttributes": [
    {
      "className": "ExampleSchema.ExampleCustomAttribute"
    }
  ]
}

🔗ECStructProperty

typeName Required

  • JSON Type: String
  • Fully qualified name {schema name}.{struct class name}.

Example:

{
  "propertyType": "StructProperty",
  "name": "ExampleStructProp",
  "typeName": "ExampleSchema.ExampleStruct"
}

🔗ECPrimitiveArrayProperty

typeName Required

  • JSON Type: String
  • One of the 10 primitive types or a fully qualified enumeration name {schema name}.{enumeration name}.

extendedTypeName Required

  • JSON Type: String

minValue Optional

  • JSON Type: Number

maxValue Optional

  • JSON Type: Number

minLength Optional

  • JSON Type: Number

maxLength Optional

  • JSON Type: Number

kindOfQuantity Optional

  • JSON Type: String
  • Fully qualified name {schema name}.{KindOfQuantity name}.

minOccurs Optional

  • JSON Type: Number

maxOccurs Optional

  • JSON Type: Number

Example:

{
  "propertyType": "PrimitiveArrayProperty",
  "name": "ExamplePrimitiveArray",
  "typeName": "ExampleSchema.ExampleEnumeration",
  "minOccurs": 7,
  "maxOccurs": 20
}

🔗ECStructArrayProperty

typeName Required

  • JSON Type: String
  • Fully qualified struct class name {schema name}.{struct class name}.

minOccurs Optional

  • JSON Type: Number

maxOccurs Optional

  • JSON Type: Number

Example:

{
  "propertyType": "StructArrayProperty",
  "name": "ExampleStructArray",
  "typeName": "ExampleSchema.ExampleStruct",
  "minOccurs": 5,
  "maxOccurs": 25
}

🔗ECNavigationProperty

relationshipName Required

  • JSON Type: String
  • Fully qualified relationship class name {schema name}.{relationship class name}

direction Required

  • JSON Type: String
  • Supported values:
    • forward
    • backward

Example:

{
  "propertyType": "NavigationProperty",
  "name": "ExampleNavigation",
  "relationshipName": "ExampleSchema.ExampleRelationship",
  "direction": "forward"
}

🔗Example Schema

{
  "$schema": "https://dev.bentley.com/json_schemas/ec/31/draft-01/ecschema",
  "name": "ExampleSchema",
  "version": "1.0.0",
  "alias": "ex",
  "label": "Example Schema",
  "description": "An example schema to showcase new ECJson format.",
  "references": [
    {
      "name": "CoreCustomAttributes",
      "version": "1.0.0"
    }
  ],
  "customAttributes": [
    {
      "className": "CoreCustomAttributes.HiddenSchema"
    }
  ],
  "items": {
    "ExampleEntity" : {
      "schemaItemType": "EntityClass",
      "modifier": "none",
      "label": "ExampleEntity",
      "description": "An example entity class.",
      "baseClass": "ExampleSchema.ExampleBaseEntity",
      "mixins": [
        "ExampleSchema.ExampleMixin",
        "ExampleSchema.ExampleMixin2"
      ]
    },
    "ExampleMixin": {
      "schemaItemType": "Mixin",
      "appliesTo": "ExampleSchema.ExampleEntity",
      "properties": [
        {
          "propertyType": "PrimitiveProperty",
          "name": "ExamplePrimitiveProp",
          "description": "An example primitive property",
          "label": "Example Primitive Property",
          "readOnly": true,
          "category": "ExampleSchema.ExamplePropertyCategory",
          "typeName": "string",
          "extendedTypeName": "Json",
          "minLength": 12,
          "maxLength": 200,
          "kindOfQuantity": "ExampleSchema.ExampleKoQ",
          "customAttributes": [
            {
              "className": "ExampleSchema.ExampleCustomAttribute"
            }
          ]
        }
      ]
    },
    "ExampleMixin2": {
      "schemaItemType": "Mixin",
      "baseClass": "ExampleSchema.ExampleMixin",
      "appliesTo": "ExampleSchema.ExampleEntity",
      "properties": [
        {
        "propertyType": "StructProperty",
        "name": "ExampleStructProp",
        "typeName": "ExampleSchema.ExampleStruct"
        }
      ]
    },
    "ExampleStruct": {
      "schemaItemType": "StructClass",
      "modifier": "sealed",
      "properties": [
        {
          "propertyType": "PrimitiveArrayProperty",
          "name": "ExamplePrimitiveArray",
          "typeName": "ExampleSchema.ExampleEnumeration",
          "minOccurs": 7,
          "maxOccurs": 20
        }
      ]
    },
    "ExampleEntity1": {
      "schemaItemType": "EntityClass",
      "modifier": "sealed",
      "baseClass": "ExampleEntity"
    },
    "ExampleTargetEntity": {
      "schemaItemType": "EntityClass",
      "modifier": "sealed"
    },
    "ExampleRelationship": {
      "schemaItemType": "RelationshipClass",
      "modifier": "sealed",
      "strength": "embedding",
      "strengthDirection": "forward",
      "source": {
        "multiplicity": "(0..1)",
        "roleLabel": "source roleLabel",
        "polymorphic": true,
        "abstractConstraint": "ExampleSchema.ExampleEntity",
        "constraintClasses": [
          "ExampleSchema.ExampleEntity",
          "ExampleSchema.ExampleEntity1"
        ]
      },
      "target": {
        "multiplicity": "(1..*)",
        "roleLabel": "target roleLabel",
        "polymorphic": true,
        "constraintClasses": [
          "ExampleSchema.ExampleTargetEntity"
        ]
      }
    },
    "ExampleKoQ": {
      "schemaItemType": "KindOfQuantity",
      "persistenceUnit": "MM",
      "precision": 3,
      "defaultPresentationUnit": "IN",
      "alternativePresentationUnits": "MM;CM"
    },
    "ExampleEnumeration": {
      "schemaItemType": "Enumeration",
      "backingTypeName": "int",
      "isStrict": true,
      "enumerators": [
        {
          "value": 1,
          "label": "None"
        },
        {
          "value": 2,
          "label": "All"
        }
      ]
    },
    "ExampleCustomAttribute": {
      "schemaItemType": "CustomAttributeClass",
      "modifier": "sealed",
      "appliesTo": "Schema|AnyProperty"
    }
  }
