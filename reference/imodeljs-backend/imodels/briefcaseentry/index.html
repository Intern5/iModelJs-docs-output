    🔗BriefcaseEntry Class

    Entry in the briefcase cache

    🔗Methods

    Name Description
    getPathKey ( ): string Public Gets the path key to be used in the cache and iModelToken

    🔗Properties

    Name Type Description
    briefcaseId🔗 number Briefcase Id
    changeSetId string Id of the last change set that was applied to the BIM.
    changeSetIndex number Index of the last change set that was applied to the BI.
    conflictError🔗 ConflictingCodesError Error set if push has succeeded, but updating codes has failed with conflicts
    fileId🔗 undefined | string File Id used to upload change sets for this briefcase (only setup in Read-Write cases)
    iModelDb IModelDb In-memory handle fo the IModelDb that corresponds with this briefcase.
    iModelId🔗 string Id of the iModel - set to the DbGuid field in the BIM, it corresponds to the Guid used to track the iModel in iModelHub
    isOpen🔗 boolean Flag to indicate if the briefcase is currently open
    isStandalone🔗 undefined | true | false Flag indicating if the briefcase is standalone or from the iModelHub
    nativeDb🔗 NativeDgnDb In-memory handle of the native Db
    openMode🔗 OpenMode Mode used to open the iModel
    pathname🔗 string Absolute path where the briefcase is cached/stored
    reversedChangeSetId undefined | string Id of the last change set that was applied to the BIM after it was reversed.
    reversedChangeSetIndex undefined | number Index of the last change set that was applied to the BIM after it was reversed.
    userId undefined | string Id of the user that acquired the briefcase.
