    🔗EditManipulator Class

    A manipulator maintains a set of controls used to modify an element.The manipulator is a new object that gets created by the handler for a supplied element.The EditManipulator controls are driven by view based point to point changes.

    🔗Methods

    Name Description
    🔗doCleanupControls(): void Abstract Called to have manipulator clean up its internal control data so that
    doCreateControls might be called again.
    🔗doCreateControls(): boolean Abstract Called to have manipulator populate its internal set of controls.
    🔗doModify(ev: BeButtonEvent, isDynamics: boolean): BentleyStatus Abstract It is expected that this is where all the work is done to update the element.
    🔗doUpdateFlashedControls(ev: BeButtonEvent): boolean Abstract Flash control(s) at the button location.
    🔗doUpdateSelectedControls(ev: BeButtonEvent | FenceParams, mode: ManipulatorSelectionMode): boolean Abstract Select control(s) that the button location is over.
    🔗doUpdateSelectedControls(path: HitDetail): boolean Abstract Select control(s) that make the most sense for the supplied hit.
    🔗hasSelectedControls(): boolean Abstract Check whether manipulator currently has any controls selected.
    🔗isDisplayedInView(vp: Viewport): boolean Abstract Return whether manipulator controls or dynamics should be shown in the supplied view.
    🔗onClick(_ev: BeButtonEvent): boolean Called on a data button event to allow controls to act on a single click.
    🔗onDoubleClick(_path: HitDetail): boolean Called when user double clicks on the manipulator's element.
    🔗onDragEnd(_ev: BeButtonEvent): void Called when a drag operation ends.
    🔗onDragStart(_ev: BeButtonEvent): boolean Called when a drag operation starts.This method is only useful for manipulators that
    wish to distinguish click - move - click from down - drag - release.
    🔗onDraw(_context: DecorateContext): void Abstract Called to display the manipulator's controls in the supplied viewport.
    🔗onGetDescription(): string Abstract Return a string suitable for display in a tool tip that describes the currently selected controls.
    🔗onKeyTransition(_wentDown: boolean, _key: number, _shiftIsDown: boolean, _ctrlIsDown: boolean): boolean Called when a keyboard key is pressed while an element is being manipulated.
    🔗onModifierKeyTransition(_wentDown: boolean, _key: number): void Called if a modifier key goes up or down while dragging.Manipulator can use this
    to cycle between different drag behaviors for a control, ex.move or copy.
    🔗onModify(ev: BeButtonEvent, context: DynamicsContext): BentleyStatus Abstract Called to show modify dynamics.
    🔗onModifyAccept(ev: BeButtonEvent): BentleyStatus Abstract Called to accept modify operation.
    🔗onModifyCancel(ev: BeButtonEvent): void Abstract Called if user aborts drag operation (reset), manipulator can cleanup anything done in onModifyStart.
    🔗onModifyStart(ev: BeButtonEvent): void Abstract Called when the modify operation is about to begin.
    🔗onNewHit(hit: HitDetail): boolean Called when manipulator is displaying controls and user clicks on the same or different geometry
    instead of on a control.
    🔗onPreModify(ev: BeButtonEvent): void Abstract When multiple manipulators are active on a selection set, only the one whose control is actually
    selected to start the modify operation is given responsibility for setting up the modification.
    🔗onRightClick(_ev: BeButtonEvent): boolean Called on right - click when not manipulating a control.One use of this event would be to
    present the user with a menu of editing options.
    🔗onSubSelection(): boolean Called when a sub entity selection effects an element that is being manipulated.
