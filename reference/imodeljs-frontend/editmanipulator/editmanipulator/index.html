🔗EditManipulator - Class

A manipulator maintains a set of controls used to modify an element.The manipulator is a new object that gets created by the handler for a supplied element.The EditManipulator controls are driven by view based point to point changes.

🔗Methods

🔗doCleanupControls PublicAbstract

doCleanupControls(): void

Called to have manipulator cleanup it's internal control data so that doCreateControls might be called again. For example after a successful drag, doCleanupControls will be called followed by doCreateControls to have the controls reflect the changes to the element.

Returns: void

🔗doCreateControls PublicAbstract

doCreateControls(): boolean

Called to have manipulator populate it's internal set of controls.

Returns: boolean - true if controls were created.

🔗doModify PublicAbstract

doModify(ev: BeButtonEvent, isDynamics: boolean): BentleyStatus

It is expected that this is where all the work is done to update the element. Using the anchor point from _OnModifyStart and the supplied button event for the cursor location, update the element data.

Parameter Type Description
ev BeButtonEvent Current button event,
isDynamics boolean Whether manipulator is being called from _OnModify or _OnModifyAccept.

Returns: BentleyStatus - SUCCESS if modify operation could be applied.

🔗doUpdateFlashedControls PublicAbstract

doUpdateFlashedControls(ev: BeButtonEvent): boolean

Flash control(s) at the button location. A flashed control is considered to be selected. Called on motion, controls should not remain selected when the button is no longer over them.

Parameter Type Description
ev BeButtonEvent

Returns: boolean - true if a control is flashed.

🔗doUpdateSelectedControls PublicAbstract

doUpdateSelectedControls(ev: BeButtonEvent|FenceParams, mode: ManipulatorSelectionMode): boolean

Select control(s) that the button location is over. Controls should remain selected until specifically de-selected.

Parameter Type Description
ev BeButtonEvent|FenceParams
mode ManipulatorSelectionMode

Returns: boolean - true if any controls remain selected.

doUpdateSelectedControls(path: HitDetail): boolean

Select control(s) that make the most sense for the supplied hit. This method supports modification tools that need to select both the element and controls from a single click and immediately start modification. The manipulator might not ever be asked to draw it's controls.

note A manipulator should always try to select at least one control.

Parameter Type Description
path HitDetail

Returns: boolean - true if any control is selected.

🔗hasSelectedControls PublicAbstract

hasSelectedControls(): boolean

Check whether manipulator currently has any controls selected.

Returns: boolean - true if manipulator has controls selected or flashed.

🔗isDisplayedInView PublicAbstract

isDisplayedInView(vp: Viewport): boolean

Return whether manipulator controls or dynamics should be shown in the supplied view.

Parameter Type Description
vp Viewport

Returns: boolean

🔗onClick Public

onClick(_ev: BeButtonEvent): boolean

Called on a data button event to allow controls to act on a single click.

note This method can be used to launch editors as it is called before _OnPreModify.

Parameter Type Description
_ev BeButtonEvent

Returns: boolean - true if event was handled and modify dynamics should not be started.

🔗onDoubleClick Public

onDoubleClick(_path: HitDetail): boolean

Called when user double clicks on the manipulator's element. (ex. edit text)

Parameter Type Description
_path HitDetail

Returns: boolean - true if manipulator handled double - click event.

🔗onDragEnd Public

onDragEnd(_ev: BeButtonEvent): void

Called when a drag operation ends.

Parameter Type Description
_ev BeButtonEvent

Returns: void

🔗onDragStart Public

onDragStart(_ev: BeButtonEvent): boolean

Called when a drag operation starts.This method is only useful for manipulators that wish to distinguish click - move - click from down - drag - release.

Parameter Type Description
_ev BeButtonEvent

Returns: boolean - true if the manipulator has handled the event.

🔗onDraw PublicAbstract

onDraw(_context: DecorateContext): void

Called to display the manipulator's controls in the supplied viewport. This is commonly done by drawing sprites, but the manipulator can choose to draw any geometry it wants.

Parameter Type Description
_context DecorateContext

Returns: void

🔗onGetDescription PublicAbstract

onGetDescription(): string

Return a string suitable for display in a tool tip that describes the currently selected controls.

Returns: string - String to describe selected controls or an empty string for no description.

🔗onKeyTransition Public

onKeyTransition(_wentDown: boolean, _key: number, _shiftIsDown: boolean, _ctrlIsDown: boolean): boolean

Called when a keyboard key is pressed while an element is being manipulated. This method is only called for a small set of specific keys presses(See below).

Parameter Type Description
_wentDown boolean true if the key was pressed, false if the key was released.
_key number One of VK_TAB, VK_RETURN, VK_END, VK_HOME, VK_LEFT, VK_UP, VK_RIGHT, VKdOWN.
_shiftIsDown boolean The shift key was down during the transition.
_ctrlIsDown boolean The control key was down during the transition.

Returns: boolean - true if the manipulator has handled the key.This will prevent further processing of the key press.

🔗onModifierKeyTransition Public

onModifierKeyTransition(_wentDown: boolean, _key: number): void

Called if a modifier key goes up or down while dragging.Manipulator can use this to cycle between different drag behaviors for a control, ex.move or copy.

Parameter Type Description
_wentDown boolean
_key number

Returns: void

🔗onModify PublicAbstract

onModify(ev: BeButtonEvent, context: DynamicsContext): BentleyStatus

Called to show modify dynamics. Expected to call doModify with isDynamics set to true, and to display the result if doModify returned SUCCESS.

Parameter Type Description
ev BeButtonEvent
context DynamicsContext

Returns: BentleyStatus - SUCCESS if modify operation could be applied.

🔗onModifyAccept PublicAbstract

onModifyAccept(ev: BeButtonEvent): BentleyStatus

Called to accept modify operation. Expected to call doModify with isDynamics set to false, and to update the element in the DgnDb if doModify returned SUCCESS.

Parameter Type Description
ev BeButtonEvent

Returns: BentleyStatus - SUCCESS if modify operation could be applied and element was updated.

🔗onModifyCancel PublicAbstract

onModifyCancel(ev: BeButtonEvent): void

Called if user aborts drag operation (reset), manipulator can cleanup anything done in onModifyStart.

Parameter Type Description
ev BeButtonEvent

Returns: void

🔗onModifyStart PublicAbstract

onModifyStart(ev: BeButtonEvent): void

Called when the modify operation is about to begin. It is the responsibility of the manipulator to keep the point in the supplied button event to use as the anchor point of the modify operation.

Parameter Type Description
ev BeButtonEvent

Returns: void

🔗onNewHit Public

onNewHit(hit: HitDetail): boolean

Called when manipulator is displaying controls and user clicks on the same or different geometry instead of on a control. Manipulator may choose to present the user with a different set of controls or cleanup. Transient manipulators need to verify that the new hit is compatible as caller has no way of checking.

Parameter Type Description
hit HitDetail

Returns: boolean - true to clear manipulator, false if current manipulator is still ok.

🔗onPreModify PublicAbstract

onPreModify(ev: BeButtonEvent): void

When multiple manipulators are active on a selection set, only the one whose control is actually selected to start the modify operation is given responsibility for setting up the modification. If called, the manipulator is responsible for setting the anchor point that will be used by all active manipulators to the center of it's selected control. This manipulator can also enable AccuSnap/AccuDraw.

note If you merely want a control to handle clicks, implement _OnClick instead.

see _OnClick

Parameter Type Description
ev BeButtonEvent Current button event, point needs to be set to center of selected control.

Returns: void - false to reject starting modify dynamics.

🔗onRightClick Public

onRightClick(_ev: BeButtonEvent): boolean

Called on right - click when not manipulating a control.One use of this event would be to present the user with a menu of editing options.

note It is up to the manipulator to check whether a control is selected or under the cursor by calling HaveSelectedControls.A manipulator could also check if the current auto - locate HitDetail is for the geometry that the manipulator was created for when not over a control.

Parameter Type Description
_ev BeButtonEvent

Returns: boolean - true if manipulator wants to do something and handled the right - click event.

🔗onSubSelection Public

onSubSelection(): boolean

Called when a sub entity selection effects an element that is being manipulated. Typically the manipulator should get the current selection state from the element and update its display as appropriate.

Returns: boolean - true if the manipulator has handled the event.
